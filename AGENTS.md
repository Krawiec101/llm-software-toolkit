# Project instructions

## Lessons learned

- For every repository change, review `docs/lessons-learned.md` before finishing the work.
- Add or update an entry when the change reveals a process failure, ambiguous instruction, missing safeguard or reusable project decision.
- Keep entries specific: record the change, the lesson and the follow-up rule that should guide future work.
- Do not add an entry for routine mechanical edits that produce no reusable lesson.
- Include the lessons-learned decision in the final change summary, even when no entry was needed.
