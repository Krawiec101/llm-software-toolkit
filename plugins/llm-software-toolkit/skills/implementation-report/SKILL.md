# Implementation Report

Read and obey `references/php-api-skill-contract.md` before creating `docs/implementation-report.md`.

Report changed files, implemented changes, generated artifacts, commands, validation results, skipped checks, residual risks, deviations from plan and an ADR evidence map. Mark every approved ADR `implemented`, `partially implemented`, `not implemented` or `blocked`; never infer success from a skipped check.

Completion criterion: every approved ADR appears exactly once with evidence, and overall success is reported only when every ADR is `implemented`.
