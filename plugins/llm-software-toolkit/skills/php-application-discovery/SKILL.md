# PHP Application Discovery

Read `references/php-api-skill-contract.md`; apply its inspection and reporting rules if files may change.

1. Inspect available project context, then collect project name, base namespace, purpose, primary use cases, business areas, rule complexity, expected 6-12 month growth, team shape only when it affects architecture, Symfony or Laravel preference, persistence need, testing preferences, MVP scope and deferred work.
2. Ask only questions whose answers materially alter those fields. Do not select architecture, libraries or versions.
3. Publish the complete discovery record, marking genuinely unknown non-material fields explicitly.

Completion criterion: every listed field is explicit and no unresolved question can materially change scope or later architecture classification.
