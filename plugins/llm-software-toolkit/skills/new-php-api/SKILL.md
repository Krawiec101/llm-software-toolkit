# New PHP API

1. Read the complete `workflows/new-php-api/WORKFLOW.md` before planning or editing. Continue when every input, stage, gate and completion condition is understood.
2. Execute its planning stages in order and stop at its explicit architecture-report gate.
3. After approval, preserve the approved report and ADRs as the source of truth for implementation and validation.
4. End with the required validation and implementation report.

Completion criterion: every accepted ADR is implemented and validated, or the output identifies the exact blocking stage and evidence.
